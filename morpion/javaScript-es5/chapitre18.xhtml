TD2: Validation de formulaire

Mise en situation

Un formulaire possède l'évènement submit qui envoie les données contenues dans le formulaire via le protocole HTTP. Si une donnée n'est pas valide, le serveur après avoir traité le formulaire doit envoyer une réponse pour l'afficher de nouveau en donnant les indications permettant de corriger la saisie.

Afin de limiter les allez-retour et le traitement côté serveur, JavaScript peut faciliter la saisie du formulaire en fournissant des renseignements sur les données attendues et empêcher son envoi s'il n'est pas rempli correctement. L’avantage est aussi d'offrir une meilleure expérience pour l'utilisateur en utilisant d'autres évènements que submit comme keyup, ou blur. En utilisant JavaScript pour valider un formulaire il ne faut pas perdre de vue que la validation se fait côté serveur, il est possible de soumettre un formulaire sans passer par une saisie et sans que JavaScript puisse empêcher son envoi.

Analyse des besoins

En vue de valider un ou plusieurs formulaires il est possible de créer un objet pour automatiser leur validation. Nous pouvons établir une liste de fonctionnalités que doit proposer cet objet.

  • Enregistre un motif pour un élément nommé
  • Enregistre un message pour un élément nommé
  • Annule l'envoi du formulaire s'il est mal renseigné
  • Peut ajouter/supprimer des messages avant l'envoi du formulaire
  • Peut ajouter/supprimer un message pendant la saisie d'un champ
  • Peut ajouter/supprimer des messages quand un champ perd son focus
  • Stylise un ou des champs mal renseignés
  • Stylise un ou des champs bien renseignés

Script

L'objet doit posséder une méthode pour enregistrer la définition des motifs et messages relatifs aux éléments nommés du formulaire et qui transiteront jusqu'au serveur lors de son envoi, cette méthode attend un nom, un motif et un message. Les conditions définies seront affectées entant que propriété à un objet à la visibilité limité nommé condition. Le module doit aussi posséder une fonction qui attend en paramètre un formulaire pour initialiser ses écouteurs. Pour le reste la logique de traitement peut avoir une visibilité privée car l'on ne souhaite pas pour l'instant accéder à d'autres fonctionnalités. Nous désirons comme au chapitre précédent pouvoir enregistrer le module entant que propriété de jQuery et chaque appel renvoie un nouvel objet afin de pouvoir plusieurs formulaires sans utiliser une méthode de reproduction d'objet comme la méthode create de Object.

Annuler l'envoi

Pour annuler l'envoi du formulaire une fonction assurant la compatibilité de l'utilisation de preventDefault peut être implémentée.

prevent function prevent(event) {
    event.preventDefault ? event.preventDefault() : (event.returnValue = false);
}
Afficher un message

Pour afficher le message d'aide à la saisie pour un élément nommé du formulaire une fonction peut créer et insérer le message avant l'élément mal renseigné.

insert function insert(message) {
    var previous = this.previousSibling;
    if ("alert" !== previous.className) {
        previous = window.document.createElement("em");
        previous.className = "alert";
        this.parentNode.insertBefore(previous, this);
    }
    previous.innerHTML = message;
}
Supprimer un message

Si un élément qui a été mal renseigné est corrigé il faut pouvoir supprimer le message d'aide à la saisie.

remove function remove() {
    var previous = this.previousSibling;
    if ("alert" === previous.className) {
        this.parentNode.removeChild(previous);
    }
}
Valider un élément

Si un élément est considéré comme valide, une méthode doit invoquer la fonction de suppression du message et affecter sa propriété className pour le styliser.

valid function valid() {
    remove.call(this);
    this.className = "true";
}
Invalider un élément

Si un élément est considéré comme invalide, une méthode doit invoquer la fonction d'insertion du message et affecter sa propriété className pour le styliser.

invalid function invalid(message) {
    insert.call(this, message);
    this.className = "false";
}
Tester la valeur d'un élément

Une fonction peut être implémentée pour tester la correspondance entre un motif et la valeur d'un élément.

test function test(pattern) {
    return pattern.test(this.value);
}
Valider la valeur d'un élément

Une fonction ayant la responsabilité d'invoquer les fonctions décrites préalablement et contenant la logique de validation doit être implémentée. Son contexte est celui d'un élément du formulaire, elle invoque valide, ou invalide et preventDefault si le test sur la valeur vaut false.

value function value(event) {
    if (!condition[this.name]) {
        return;
    } else if (!test.call(this, condition[this.name].pattern)) {
        invalid.call(this, condition[this.name].message);
        prevent(event);
    } else {
        valid.call(this);
    }
}
Contexte d'un champ

Nous allons implémenter une fonction field. Sa responsabilité est d'invoquer une fonction fournie en paramètre en lui rattachant le contexte d'un élément afin de ne pas répéter des instructions proches, cette fonction fournie une couche d'abstraction à l'invocation des fonctions. Elle peut être utilisée pour affecter différents écouteurs sur les éléments ou pour valider les éléments lorsque que l'évènement submit est émis.

field function field(func, args) {
    for (var i = 0,
             tag = ["input", "textarea"],
             tagLength = tag.length; i < tagLength; i++) {
        for (var j = 0,
                 field = this.getElementsByTagName(tag[i]),
                 fieldLength = field.length; j < fieldLength; j++) {
            func.apply(field[j], args);
        }
    }
}
Écouteurs

Pour ajouter les écouteurs aux champs du formulaire et au formulaire, nous pouvons créer deux fonctions mais leur présence peut être évitée en les ajoutant directement dans le constructeur de l'objet au sein du squelette présenté. Pour des soucis de lisibilité des fonctionnalités du module elles sont implémentées.

fieldListeners et formListeners function fieldListeners(condition) {
    this.onblur = function (event) {
        value.apply(this, [event || window.event, condition]);
    };
    this.onkeyup = function () {
        if (!condition[this.name]) {
            return;
        } else if ("true" !== this.className
               && test.call(this, condition[this.name].pattern)) {
            valid.call(this);
        } else if ("true" === this.className
               && !test.call(this, condition[this.name].pattern)) {

        }
    };
}

function formListeners(condition) {
    this.onsubmit = function (event) {
        field.call(this, value, [event || window.event, condition]);
    };
}

Correction

En invoquant formValidation un nouvel objet est créé, la méthode setCondition défini les conditions de validation de ses champs. Nous utilisons des expressions régulières et observons que les classes de caractères pour définir un ensemble de caractères accentués utilisent la notation hexadécimale. Concernant le motif décrivant l'adresse mail, il faut savoir qu'un email peut ne pas correspondre avec ce motif et il est déconseillé d'utiliser un motif pour vérifier une adresse email, sa syntaxe définie par la RFC 822 ne peut pas être vérifiée à 100% par une expression régulière.

window.onload = function () {

    var form = this.formValidation(document.getElementById("form"));
    form.setCondition(
        "firstname",
        /^[A-Z\xc1-\xdd]{1}[A-Za-z\xc1-\xff-\s]{2,31}$/,
        "Le prénom commence par une majuscule et possède de 3 à 32 caractères alphabétiques");
    form.setCondition(
        "lastname",
        /^[A-Z\xc1-\xdd]{1}[A-Za-z\xc1-\xff-\s]{2,63}$/,
        "Le nom commence par une majuscule et possède de 3 à 64 caractères alphabétiques");
    form.setCondition(
        "company",
        /(^$|^[A-Za-z0-9\xc1-\xff-\s]{3,32}$)/,
        "Le société possède de 3 à 32 caractères alphanumériques");
    form.setCondition(
        "email",
        /^([a-z0-9._-]{3,64})+@([a-z0-9.-]{3,64})+\.[a-z]{2,8}$/,
        "L'adresse email n'est pas valide");
    form.setCondition(
        "phone",
        /(^$|^(\+33\s)?0+[0-9]{1}([\.-\s]?[0-9]{2}){4}$)/,
        "Le numéro de téléphone n'est pas valide");
    form.setCondition(
        "message",
        /(.|\n){10,300}/,
        "Le message possède de 10 à 300 caractères"); 
};

    function prevent(event) {
        event.preventDefault ? event.preventDefault() : (event.returnValue = false);
    }

     * Insert message
     * @function insert

     * @param {string} message Message d'aide

        var previous = this.previousSibling;
        if ("alert" !== previous.className) {
            previous = window.document.createElement("em");
            previous.className = "alert";

        }
        previous.innerHTML = message;
    }

     * Remove message
     * @function remove
     * @access private
     */

        if ("alert" === previous.className) {

        }

     * Invalid element

     * @access private
     * @param {string} message Message d'aide

        this.className = "false";
    }

    /**
     * Valid element
     * @function invalid
     * @access private
     */

        remove.call(this);
        this.className = "true";
    }

    /**
     * Test element value
     * @function test
     * @access private
     * @param {RegExp} pattern Motif
     */

        return pattern.test(this.value);
    }

    /**
     * Valid element value
     * @function value
     * @access private
     * @param {Event} event Evènement émis
     * @param {Object} condition Condition de validation

        if (!condition[this.name]) {
            return;
        } else if (!test.call(this, condition[this.name].pattern)) {
            invalid.call(this, condition[this.name].message);
            prevent(event);
        } else {
            valid.call(this);
        }
    }

     * Apply field to func
     * @function field
     * @access private
     * @param {func} func Fonction à appliquer
     * @param {array} args Arguments à appliquer
     */

        for (var i = 0,
                 tag = ["input", "textarea"],
                 tagLength = tag.length; i < tagLength; i++) {

                     field = this.getElementsByTagName(tag[i]),
                     fieldLength = field.length; j < fieldLength; j++) {
                func.apply(field[j], args);
            }
        }
    }

        };

            if (!condition[this.name]) {
                return;
            } else if ("true" !== this.className

                valid.call(this);
            } else if ("true" === this.className
                   && !test.call(this, condition[this.name].pattern)) {

Le formulaire est enrichi et l'utilisateur peut le compléter plus facilement grâce aux messages d'aide à la saisie affichés lors de son interaction avec lui. Quand le formulaire est soumis, les champs dont l'attribut name est présent dans les conditions sont vérifiés, s'ils sont valides ils sont stylisés en vert, s'ils sont invalides, un message d'aide à la saisie s'affiche et ils sont stylisés en orange. Quand un élément est renseigné, au moment du keyup, si sa valeur est valide le champ est stylisé, s'il est valide puis suite à une modification devient invalide, le message s'affiche et il est stylisé invalide. Au moment où un champ perd son focus sa valeur est vérifiée et il devient soit valide soit invalide. Cet exercice a proposé de réviser la manipulation des contextes d'exécution, les évènements et la compatibilité de Event sur les navigateurs et la modification du DOM.
