Chapitre 07: Le Document Object Model

Vers le DHTML

Les bases du langage ont été décrites dans les chapitres précédents et nous permettent maintenant d'apprendre à utiliser JavaScript pour interagir avec un document HTML, XHTML ou XML. Ces documents ont en commun un modèle: le Document Object Model.

Introduction

Le DOM

Le Document Object Model est une convention pour représenter un document HTML, XHTML ou XML et pour pouvoir interagir avec. Il est le lien entre un document et un langage pouvant le manipuler.

1998

Le DOM de niveau 1 est publié et décrit un document sous forme d'arbre composé de nœuds à partir de son balisage. Il offre une série de fonctions permettant de parcourir et de modifier les éléments du DOM.

2000

Le DOM de niveau 2 propose un système d'événements génériques qui permet l'enregistrement des gestionnaires d'événements et fournit des informations contextuelles de base pour chaque événement. Le DHTML prend tout son sens il est possible d’interagir avec le document via une approche évènementielle de la programmation.

2004

Le DOM de niveau 3 enrichit l'accès aux éléments du DOM avec l'implémentation des expressions XPath, un chemin structuré rend plus facile la navigation dans le document. Les évènements sont aussi enrichis par l'implémentation de keydown, keyup et keypress.

DHTML et XHTML

Depuis le DOM de niveau 3 le DHTML et le XHTML ont rendu JavaScript incontournable comme langage pour dynamiser un document notamment via les évènements. Le DHTML pour Dynamic Hypertext Markup Language est un document qui est manipulé dynamiquement avec les ressources présentes à son chargement, les informations qui lui sont injectées sont stockées dans le script, un tel document n'utilise pas de requêtes pour s'enrichir. Le XHTML pour Extensible Hypertext Markup Language s'appuie principalement sur l'objet XMLHttpRequest ou l'objet ActiveXObject permettant l'échange de données via le protocole HTTP pour dynamiser le document.

L'objet Window

window

Le Document Object est une propriété de l'objet window. Nous allons nous attarder sur l'objet window et la gestion de ses propriétés.

Lorsqu’une variable globale est créée elle est affectée entant que propriété de l'objet window.

var globale = true;

(function () {
    var locale = true;
    implicite = true;
})()

// true
console.log(window.globale);
// undefined
console.log(window.locale);
// true
console.log(window.implicite);

delete

Une variable créée avec le mot var tout comme une propriété intégrée à l'objet window ne peut pas être supprimée avec l'opérateur delete comme pour un objet classique. Il faut remarquer qu'il est possible d'utiliser delete avec une variable implicite. Bien que rattachée à l'objet window, une variable implicite n'est pas une VariableEnvironment mais une LexicalEnvironment qui est rattachée par défaut à l'objet global soit à window. Une variable d'environnement ne peut pas être supprimée.

var globale = true;
implicite = true;

delete window.globale;
delete window.implicite;

// true
console.log(window.globale);
// undefined
console.log(window.implicite);
window.document

L'objet document, propriété de l'objet window contient le document modèle. Il ne peut pas être supprimé car il est une variable d'environnement mais il ne peut pas être redéfini non plus.

// object
console.log(typeof window.document);

window.document = true;
var document = true;
document = true; 

// object
console.log(typeof window.document);

Les nœuds du document

Les nœuds

Il existe 7 types de noeuds qui ne sont pas dépréciés, le dom n'est pas constitué uniquement d'éléments.

Constante Valeur Représentation
ELEMENT_NODE 1 Une balise
TEXT_NODE 3 Du texte
PROCESSING_INSTRUCTION_NODE 7 Une instruction XML
COMMENT_NODE 8 Un commentaire
DOCUMENT_NODE 9 Un document
DOCUMENT_TYPE_NODE 10 Un doctype
DOCUMENT_FRAGMENT_NODE 11 Un fragment

Le document et ses nœuds possèdent des propriétés et des méthodes nous permettant d’interagir avec le DOM, d'accéder et de modifier ses nœuds, ce qui est présenté au chapitre suivant.
